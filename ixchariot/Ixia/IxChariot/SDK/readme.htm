About this Release

Thank you for choosing IxChariot.

 

The IxChariot product family is the industry-leading traffic generator and

decision support tool emulating real-world application data without the need to

install and maintain extensive client/server networks. Incorporating the

IxChariot Console, Performance Endpoints and Application Scanner, the IxChariot

family offers thorough application assessment and device testing by emulating

hundreds of protocols across thousands of network endpoints. IxChariot

provides the ability to confidently predict the expected performance

characteristics of any application running on your wired and wireless networks.

Contents of the API

This SDK contains the headers and libraries that are required to build a C program that uses the IxChariot API. It also includes sample programs in C and Tcl. The API Guide for IxChariot is included with the IxChariot online help and is also available in PDF format in the Books directory on the IxChariot CD-ROM.  The structure of the directories is as follows:

 

INCLUDE

 

IxChariot API C header file

LIB

 

IxChariot API C library file

SAMPLES

 

 

C 

 

Zip file containing C programming examples

TCL

 

Zip file containing Tcl programming examples

Building the C Programs

The makefile included with the C examples can build all of the example programs or just one specific target. The makefile assumes the INCLUDE and LIB environment variables are set correctly for your compiler. The makefile includes all the compile and link options required for the following compilers:

 

    IBM VisualAge for C++

    Microsoft Visual C++ (this is the default)

    Watcom C/C++

 

We suggest that you copy the C, H, and LIB files to a single directory before executing the makefile. The examples can be built with the following command: nmake [target]

 

Running the Tcl scripts

To run the Tcl scripts from the command line, make sure that the files with the .tcl extension are associated with TCLSH.EXE, not WISH.EXE.

The scripts should be run from an MS-DOS command prompt in order to see their results after the script completes and exits.

We have tested successfully with Tcl version 8.4.
