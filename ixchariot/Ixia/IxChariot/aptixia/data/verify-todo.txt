====================
= Comment Legend
====================

DONE Before the .xml file name
name means it's incomplete

## = Property Validator comment
## = needs a property validator i.e. a Validate_PROPNAME method in the node
that contained the property OR the metaschema element needs options to enforce
a valid value

##?? = needs a property validator i.e. a Validate_PROPNAME method in the node
that contained the property. TBD what exactly the checking should be

#! = Node Verifier comment. 
a Verifier override is needed in the PropertyNode(list)
for pre-test validation of the node. cross checking is done here 
or in a ancestor node Verifier when needed

#None = no work needed

#Cannot-determine = needs to be reviewed by another (expert) on the node
