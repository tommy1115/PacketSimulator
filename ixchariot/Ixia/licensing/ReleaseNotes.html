About This Release 

Ixia Licensing 2.20 SP2 is a minor releases of Ixia Licensing. It includes 
several new features and bug fixes.


In most circumstances, IxLicensing is packaged with and installed with an 
application. The list of applications packaged with this version of Ixia 
Licensing, as of the date of these release notes, is:

    - IxChariot 6.25
    - IxVerify 8.25
    - IxProfile 1.15

    
There are cases, however, in which this separately packaged version of Ixia
Licensing may be used. These include:

    - In order to install a stand-alone license server for IxChariot, IxVerify or
      IxProfile, which do not normally install a license server.
      
    - In order to install an additional license server.
    
    - To utilize the changes and fixes associated with this release for other 
      software releases. This function is normally performed by Ixia support 
      personnel.
   
         
Important Changes
-----------------
The important changes included with this release are:

    1. Remove usage of NIC MAC addresses in forming the unique "node ID" 
       associated with a system. This value has been problematic, changing with 
       replacement of NIC cards and use of wireless networks.
       
    2. Chassis locked applications may now be used via terminal server.
    
    3. Internal Ixia licenses may now be upgraded. In the past, internal licenses 
       were issued only for the current software release. Newly issued internal 
       licenses will now last for 180 days and may be used for any version of 
       software, although the LRU's "Update" function must be applied. Older 
       internal licenses should be allowed to expire.
       
    4. A licensing diagnostic is included in the IRU. See "Diagnostic Usage"
       below.

   
Diagnostic Usage
----------------
A licensing management diagnostic is now included in the IRU and its usage is 
described in the License Management Users Guide included with this package. It 
is accessed from the IRU's "Manage Licenses" screen, using the Tools>Diagnostic 
menu option. It may be used by customers and support personnel, alike. 


Use of IxChariot on Ixia Chassis
--------------------------------
This version of license management is also useful If IxChariot is installed on a 
chassis (or host with an Ixia Demo Server) that is used with other licensed 
application (IxVPN for example). This installs several additional licensing 
components needed for the other applications.

What's New in Ixia Licensing (Since 2.10)

Features

License Management
The use of NIC MAC addresses in the formation of the system node ID has been removed.
Chassis locked applications may now be used through Microsoft Remote Terminal Services.
Internal Ixia licenses may now be upgraded to new application versions with the same license during the licenses maintenance period. Only an LRU Update is required.

Installation Notes 

Removal of the use of NIC MAC addresses in forming the unique "node ID" for the 
system has the effect of invalidating all licenses installed with License 
Management version 2.10 or earlier. This is because licenses are locked to a 
particular node ID, the algorithm by which the node ID is calculated has been 
changed, so the current node ID no longer corresponds to existing licenses. 
Normally, users who install a new version of software may use the IRU's "Update" 
function to obtain an updated license; this will not work in this case. It is 
necessary for the user to use the IRU's "Deregister" function followed by the 
"Register" function. 

REMEMBER, ALL LICENSES ASSOCIATED WITH THE NODE (CHASSIS OR PC) MUST BE 
DEREGISTERED AND REGISTERED AFTER INSTALLATION OF LICENSE MANAGEMENT 2.20.

IxChariot, IxProfile and IxVerify users will find that these products will
automatically perform the deregister/register function.

Chassis locked applications may now use terminal server. This may be accomplished 
for an already issued license by using the LRU's "Update" function.
