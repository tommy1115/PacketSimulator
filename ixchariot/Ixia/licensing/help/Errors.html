Appendix B: Ixia Registration Utility (IRU) Errors and Troubleshooting

This appendix provides basic licensing troubleshooting information, and lists various errors that can occur during the registration, de-registration, or update process.

The following sections cover the troubleshooting and error messages for licensing:

    • General Problems and Troubleshooting on page B-2
    • Error Messages on page B-8

General Problems and Troubleshooting

The following sections outlines some common issues users may encounter when dealing with the Ixia licensing software. This section is not meant to be all inclusive.

For more detailed instruction for coping with licensing issues, see the FAQ and troubleshooting guide on the licensing section of the Ixia website.

Proxy Servers and Firewalls

If licenses are installed on a network that uses either proxy servers or firewalls, it may be necessary to do an offline registration, even if the Installation Host can reach the Internet.

Follow the procedures for offline registration, de-registration, and update provided in the following sections:

    • Offline License Registration – Step by Step on page 2-10
    • Offline License De-registration – Step by Step on page 3-10
    • Offline License Update – Step by Step on page 4-10
Note. If the system to which the license is being installed uses SOCKS version 5, it is not necessary to configure special proxy settings.

Linux Firewall

Some applications that run on the Linux operating system require that the Firewall be turned off for licensing to work correctly.

To disable the Linux Firewall, enter one of the following commands: 

service iptables stop

or 

/etc/init.d/iptables stop

Versions of Linux Supported

Ixia licensing currently supports Red Hat Linux version 9.0. Other versions of Linux are not fully supported.

Licensing Services

Once licensing is installed, the license server and Node ID generation run as Windows services. If there is a problem licensing or using a product, first check to see that the license services are running. These services are:

    • IxiaLicenseServer
    • noded

Disk ID Reset Utility

The utility is run at the time of the license installation. A dialog similar to the one shown in Figure B-1 is displayed:

Figure B-1. Set Disk ID Warning

Select OK to continue with the change in the disk ID number, select Cancel to retain the old disk ID. When finished, a dialog similar to figure is shown.

Figure B-2. Set Disk ID Complete

Select OK to complete the process.

The old disk ID number is stored in a file called restorediskid.bat. By running this batch file, the user can restore the old disk ID.

Note. Changing the disk ID requires that you re-register all licensed applications on the chassis. To assist in this process, a script is included with Licensing that de-registers and re-registers all existing Ixia licenses automatically. The utility is named UpdateUtil.exe and is located at C:\Program Files\Ixia\Licensing.
This utility is automatically run if the Disk ID is reset.

License Not Found Error

Most Ixia products require a license to function. When a valid license cannot be found, the Licensing software will generate an error message. I

First, check the IRU and run the diagnostics application, as described in Appendix C, Licensing Diagnostic Utility.

If the IRU hangs during the diagnostics application and does not complete:

  1. Open a command window using the Start menu.
  2. At the prompt enter the following command:

    3. netstat -an.

  3. Check to see if ports 4501 and 27000 – 27009 are in a LISTENING. state. If not, investigate if these ports could be blocked on the chassis, PC, or in the network along the traffic path.
  4. Open a web browser and enter the following URL:

    5. http://localhost:4501

    If the code text is returned, the port is open. If an error page is returned, the port is blocked.

  5. Put the chassis IP in exceptions of proxy server.

These efforts should make the license server work. Make sure that IxExplorer`s proxy settings are set to Auto-detect Proxy and Bypass For Local Addresses.

It is best to run IxExplorer first before other applications, to establish the connections to the licensing software.

Notes. Prior to IRU version 2.60, ports 4501 and 27000 - 2700 were required to be open. If these ports are blocked then you must open them on the chassis, client PC, or network so the client PC can communicate with the IRU software ( even if the license resides in the IRU on the client).
With IRU version 2.60 and higher, the customer can manually select which ports are desired for the licensing traffic to communicate on. See IxiaLM Port Configuration on page 6-14 for additional details.

Other common causes can be:

    • Changing the chassis name (the default is the SN). In this case, change the name of the chassis back to the name that the license was registered under.
    • Manually moving the license from one node to another. A forced deregistration (done by Ixia technical support) may be required if the license on the node doesn't match the one in the Ixia License Database.
    • Changing the system date. Since many files will have a timestamp change by the system, changing the date back won't resolve the problem. Often the solution requires all the changed files to be manually "touched" with the correct date. In the case where the number of files exceeds the allowed time, the only resolution is to reformat the hard-drive. For a chassis, this requires a "ghosting" procedure. For a PC, the customer must use another PC, or have the IT dept reformat the hard-drive.

Error Messages

The following section lists common error messages that the user may encounter while using the Ixia licensing system. There are two tables:

    • Registration Server Error Codes on page B-8 – Lists problems or conditions that involve the Ixia Registration Server.
    • IRU and noded Error Codes on page B-9 – Lists problems or conditions that involve the installation process on the customer site.

The tables also provide brief troubleshooting notes.
Table B-3. Registration Server Error Codes
Error Number
Description
Action
1001
Invalid Registration Number or Password
Please verify the registration number and password from the email sent from Ixia and try again. Either the registration number or password entered is incorrect.
1002
License Already in Use
A license has already been issued for this registration. Please de-register before trying to register again. There can be no more than one active license for each registration.
1003
Internal Server Error
A severe problem occurred with the Ixia Registration server. Please try again later.
1004
Invalid Deregistration Code
The de-registration code is incorrect. The user has made a mistake while copying the de-registration code from the LRCU to the website. Please provide the correct de-registration code.
If this error has occurred for online registration then the keys are out of sync on the LRCU and the registration server.
1006
Registration is No Longer Valid
This registration number can no longer be used. The registration was invalidated manually buy an authorized Ixia personnel because of RMA or incorrect data entry. Please contact Ixia to get the new registration number.
1010
This registration is already in use with an older version of the software
This registration is already in use with an older version of the software. The user is has not de-registration from the Legacy Registration Server and is trying to register the new version. Please de-register the old version and try again.
1012
Invalid Data Format: Node ID should have the format xxxxxx-xxxxxx-xxxxxx-xxxx
The Node ID was incorrectly entered. Try cutting and pasting the Node ID from the IRU instead of typing it.

Table B-4. IRU and noded Error Codes
Error Number
Description
Action
1200
The Registration Did Not Match a License File on the License Server
This occurs during de-registration or update. The customer is trying to de-register/update on the wrong machine. Please check the registration number with the chassis or workstation name to which the license applies, and/or the name of the license server and try again.
1201
Ixia Licensing Daemon Was Not Found on Target Machine
This happens either when the noded service is not running or the machine name is incorrect. Please ensure that:
  • the network is not down
  • the Node name or IP for the noded server is correct
  • the noded service is running on the target machine. Restart noded service if it's not running
  • the proxy setting is correct. See Proxy Servers and Firewalls on page B-2 for more information
1202
Invalid Registration Number or Password
This occurs when a bad password is entered in offline mode. Please verify the registration number and password from the email sent to from Ixia, and verify that the IRU is connecting to the right registration server.
1203
Invalid Registration Number
This occurs when the registration number format is incorrect. Please verify the registration number from the email sent from Ixia and try again.
1206
The License Corresponding to This Registration Number Has Not Been Registered Yet
This occurs when the user tries to de-register/upgrade a license without registering it first. Please ensure that the license is registered, and that this is the correct registration number for a license that is to be de-registered, re-registered, or upgraded.
1209
Invalid Password
Please verify your password from the email sent to you earlier and try again, and check that the password was input correctly.
