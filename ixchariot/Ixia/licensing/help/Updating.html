Chapter 4: Updating Licenses

Introduction

Ixia's products are tied to their maintenance contracts. A product can be updated without updating it's license as long as it's maintenance contract is active. Once the maintenance contract is renewed, then the product license must be updated in order to update the product to a new version.

Before starting the license update process, make sure that the following information is available:

  1. Registration number and password for the license – this comes as an e-mail from Ixia to the Lab Manager and should have been forwarded to the person in charge of the installation. The source of this e-mail and its contents are described in Before Starting Registration on page 2-1.
  2. Workstation or Ixia Chassis name – this is the name of the computer that uses the license.
  3. License Server name – if the lab will use a central License Server, the Lab manager should have provided its name or IP address.

Online License Update – Step by Step

Make sure the network connectivity requirements discussed in Installation Scenarios are understood, and that the necessary information discussed above in Introduction is available.

If the Installation Host is connected to the Internet (and to the lab network as well), an online update can be performed. The steps in the online update process are:

    • Open the Ixia Registration Utility
    • Step 1: Enter Registration Number and Password
    • Step 2: Enter a Chassis or Workstation Name
    • Step 3: Enter License Server Name
    • Step 4: Save the License

If the Installation Host is not attached to the Internet, follow the instructions in Offline License Update – Step by Step.

Note. The license update process can also be initiated from the License Management dialog, as described in Chapter 6, Managing Licenses.

Figure 4-1 will help keep track of the steps in the online registration process. It is repeated at each step with the current operation highlighted.

Figure 4-1. Online Update

Note. Depending on whether the license is being installed to a computer or a chassis, the wording in the IRU dialogs is slightly different.

Open the Ixia Registration Utility

When the IRU is opened, the screen shown in Figure 4-2 is displayed. The IRU allows the user to select various registration operations.

To open the IRU manually, select Start > Programs > Ixia > Licensing > IRU from the Start menu.

Figure 4-2. Ixia Registration Utility Main Dialog

    • Select the Update radio button.
    • Select the Start button.
Note. The other radio button operations are described in the following chapters:
  • Registration – Chapter 2, Registering Licenses
  • De-registration – Chapter 3, De-registering Licenses.
  • Manage Licenses – Chapter 6, Managing Licenses.

Step 1: Enter Registration Number and Password

After selecting the Update radio button and then selecting Start, the first screen displayed is shown in Figure 4-3. If the Installation Host is not attached to the Internet, the screen shown in Figure 4-9 is displayed. Follow the instructions in Offline License Update – Step by Step.

Figure 4-3. Online Update Step 1 –

Registration Number

    • Enter the Registration Number and Password exactly as specified in the Ixia generated e-mail. The source and contents of this e-mail are discussed in Introduction and in What is License Management?.
    • Press the Next button.

Step 2: Enter a Chassis or Workstation Name

Notes. This screen will not appear if a floating license is being updated, as floating licenses are located on a stand-alone License Servers. The process immediately jumps to Step 3: Enter License Server Name instead.
If the license being updated is on the Installation Host, then this step is skipped. The update process has automatically found the license file. Proceed to Step 4: Save the License on page 4-8.

If the license is associated with a different chassis/workstation, then a screen similar to Figure 4-4 is displayed.

Figure 4-4. Online Update Step 2 – Ixia Chassis or Workstation Name

    • Enter the host name or IP address of the Ixia chassis or workstation that will use the licensed software. Localhost is the default and refers to the Installation Host.
    • Look at the Product Features box and verify that the features listed correspond to the software or license being updated.
    • Press the Next button.

Step 3: Enter License Server Name

Notes. If a License Server is used, the Lab Manager should provide its name. The use of a License Server is discussed in more detail in Appendix A, License Management Planning Guide.
If the license for the product is found on the chassis or workstation specified in Step 2: Enter a Chassis or Workstation Name, then this step is skipped (proceed to Step 4: Save the License).
This step immediately follows Step 1: Enter Registration Number and Password if a floating license is being updated.

A License Server is an optional computer used to manage licenses in a lab. Figure 4-5 displays the dialog where the License Server is specified.

Figure 4-5. Online Update Step 3 – License Server

    • The above screen should only appear if a License Server is used (and the chassis or workstation entered in Step 2: Enter a Chassis or Workstation Name did not contain the license being updated). If a License Server is not being used, and this screen appears, press the Back button and specify the correct chassis or workstation.
    • Enter the name of the License Server in the Server name or IP address field.
    • Press the Next button.

Step 4: Save the License

Note. This step may immediately follow Step 1: Enter Registration Number and Password or Step 2: Enter a Chassis or Workstation Name, depending on the configuration of the test environment.

All the necessary information has been entered. A screen similar to the one shown in Figure 4-6 is displayed.

Figure 4-6. Online Update Step 4 –

Reregister License File

    • Verify the information shown.
    • Press the Next button to update the license.

After the license has been updated, a Receipt is displayed, similar to the one shown Figure 4-7. This is a record of the license's update. Save the information as a file or a make a printout.

Figure 4-7. Online Update Step 5 – Print Receipt

    • To save the receipt to disk, press the Save Receipt... button and designate a file.
    • To make a printout of the receipt, press the Print Receipt... button and select a printer.
    • Press Finish to exit the update process.

The updated license is now available for use on the chassis or workstation. If it is a single node-locked license, it can only be used on that particular chassis or workstation.

Offline License Update – Step by Step

The steps in the offline installation process are:

    • Step 1: Start Offline Update
    • Step 2: Enter Registration Number and Password
    • Step 3: Enter a Chassis or Workstation Name
    • Step 4: Enter License Server Name
    • Step 5: Uninstall the License
    • Step 6: Move to the Online Host
    • Step 7: Online – Connect to the Ixia Registration Website
    • Step 8: Online – Enter De-registration Code
    • Step 9: Online – Select Product Version
    • Step 10: Online – Save the License File
    • Step 11: Install the License File

Note that during a license update, the license is first de-registered, then reregistered with current product and workstation/chassis information.

Figure 4-8 will help keep track of the steps in the offline registration process. It is repeated at each step with the current operation highlighted.

Figure 4-8. Offline Update

Note. Depending on whether the license is being installed to a computer or a chassis, the wording in the IRU dialogs is slightly different.

Step 1: Start Offline Update

Open the IRU as described in Open the Ixia Registration Utility. The first screen displayed is shown in Figure 4-9. This message indicates that the Installation Host cannot reach the Ixia web-site via the Internet. If the Installation Host should be connected, use the following steps to diagnose the problem.

If the Installation Host should be attached to the Internet

    • Use a browser to go to any non-Ixia Internet page. If this fails, then there is a problem within the Installation Host.
    • Use the browser to access Ixia's registration server at http://licensing.ixiacom.com. If this fails, then it might be the case that Ixia's server is down. Contact Ixia at the appropriate support center to confirm.
    • If the Installation Host can access Ixia's registration server's web page, and the screen shown in Figure 4-9 still appears after restarting the license installation process, then either proceed with the steps in this section, or contact Ixia at the appropriate support center for assistance.

To proceed with offline license update

The dialog shown in Figure 4-9 is displayed when the Installation Host is not connected to the internet.

Figure 4-9. Offline Update Step 1 – Notice for Offline Registration

    • Find an additional computer that is connected to the Internet (the Online Host). It is necessary to transfer small amounts of data between the Online Host and the Installation Host (this machine). A removable USB drive or a common e-mail system are two options.
    • Press the Next button.

Step 2: Enter Registration Number and Password

The next screen displayed is shown in Figure 4-10.

Figure 4-10. Offline Update Step 2 – Registration Number

    • Enter the Registration Number and Password exactly as specified in the Ixia generated e-mail. The source and contents of this e-mail is discussed above in Introduction on page 4-1 and in What is License Management? on page 1-2.
    • Press the Next button.

Step 3: Enter a Chassis or Workstation Name

Notes. This screen will not appear if a floating license is being updated, as floating licenses are located on a stand-alone License Servers. The process immediately jumps to Step 4: Enter License Server Name instead.
If the license being updated is on the Installation Host, then this step is skipped. The update process has automatically found the license file. Proceed to Step 5: Uninstall the License.

If the license is associated with a different chassis or workstation, then the next screen is shown in Figure 4-11.

Figure 4-11. Offline Update Step 3 – Ixia Chassis or Workstation Name

    • Enter the host name or IP address of the Ixia chassis or workstation that will be used with the licensed software. Localhost is the default and refers to the Installation Host.
    • Look at the Product Features box and verify that the features listed correspond to the software or license being updated.
    • Press the Next button.

Step 4: Enter License Server Name

Note. If the license for the product is found on the chassis/workstation specified in Step 3: Enter a Chassis or Workstation Name, then this step is skipped (proceed to Step 6: Move to the Online Host).
This step immediately follows Step 3: Enter a Chassis or Workstation Name if a floating license is being updated.

A License Server is an optional computer used to manage licenses in a lab. Figure 4-12 displays the dialog where the License Server is specified.

Figure 4-12. Offline Update Step 4 – License Server

    • The above screen should only appear if a License Server is used (and the chassis or workstation entered in Step 3: Enter a Chassis or Workstation Name did not contain the license being updated). If a License Server is not being used, and this screen appears, press the Back button and specify the correct chassis or workstation.
    • If a License Server is used in the Test Lab, enter its name in the Server name or IP address field.
    • Press the Next button.

Step 5: Uninstall the License

Notes. This step may immediately follow Step 2: Enter Registration Number and Password or Step 3: Enter a Chassis or Workstation Name, depending on the configuration of the test environment.

All the necessary information has been entered. A screen similar to the one shown in Figure 4-13 is displayed.

Figure 4-13. Offline Update Step 5 – De-register License File

    • Verify the information shown.
    • Press the Next button to uninstall the license.

Step 6: Move to the Online Host

At this point, a screen similar to that shown in Figure 4-14 is displayed.

Figure 4-14. Offline Update Step 6 – Move to Online Host

    • The information on the screen is necessary to complete the update from the Online Host. Either:
      • Print it out using the Print... button, or
      • Save it to disk using the Save... button. This file can be copied to a removable USB drive or e-mailed to the Online Host.
    • Move to the Online Host, start a browser and enter http://licensing.ixiacom.com.
    • Follow the instructions in:
      • Step 7: Online – Connect to the Ixia Registration Website
      • Step 8: Online – Enter De-registration Code
      • Step 9: Online – Select Product Version
      • Step 11: Install the License File
    • Do not press the Next button in this dialog yet.

Step 7: Online – Connect to the Ixia Registration Website

The first online screen displayed is shown in Figure 4-15.

Figure 4-15. Offline Update Step 7 – Ixia Registration Website

    • Select the appropriate option. The links redirects the browser to a the screen shown in Figure 4-16.

      • Figure 4-16. Offline Update Step 7 – Select Action

    • Select the Update option.

Step 8: Online – Enter De-registration Code

A webpage similar to the one shown in Figure 4-17 is displayed.

Figure 4-17. Offline Update Step 8 – Enter De-registration Code

    • Enter the De-registration Code exactly as shown in Figure 4-14. It is listed directly above the Node ID. In this example, it is jmPBBysyxeBQIOU3EGPAQ==.
    • Press the Go button.

Step 9: Online – Select Product Version

The screen shown in Figure 4-18 displays the registration information and the possible version numbers of the product being licensed.

Figure 4-18. Offline Update Step 9 – Product Version

    • Verify the Registered To and Product/Features fields to ensure that this is the correct license.
    • Select the version number that matches the upgraded software or product. If necessary, the Keep Current License button can be selected to abort the update.
    • Press the Get License File button.

Step 10: Online – Save the License File

At this point, the actual Ixia license is downloaded to the computer connected to the internet. A dialog similar to that shown in Figure 4-19 is displayed.

Figure 4-19. Offline Update Step 10 – Download License File

    • Press the Save button and select a place to save the license file.
    • Transfer this saved file over to the Installation Host via a removable USB drive or e-mail (if the Installation Host has e-mail capabilities).

      • If a License Server is being used, remember to save the file to the License Server (if it is a different chassis or workstation than the Installation Host). The recommended directory is C:\Program Files\Ixia\licensing\licenses.

Step 11: Install the License File

Return to the Installation Host.

    • Press the Next button. A screen similar to the one shown in Figure 4-20 is displayed.

      • Figure 4-20. Offline Update Step 11 – Select License File

    • Enter the filename of the license file copied from the Online Host. The recommended default location is C:\Program Files\Ixia\licensing\licenses. This was the last step in Step 10: Online – Save the License File. Use the Browse... button to help locate the file.
    • Press the Next button to install the license.

After the license has been updated, a Receipt is shown, for example as in Figure 4-21. This is a record of the license's update. Save the information as a file or a make a printout.

Figure 4-21. Offline Update Step 12 – Receipt

    • To save the receipt to disk, press the Save Receipt... button and designate a file.
    • To make a printout of the receipt, press the Print Receipt... button and select a printer.
    • Press Finish to exit the update process.

The updated license is now available for use on the chassis or workstation. If it is a single node-locked license, it can only be used on that particular chassis or workstation.
