Appendix A: License Management
Planning Guide

Introduction

This document is intended for use by Lab Managers of Ixia equipment, specifically those responsible for the deployment and installation of Ixia software products.

Ixia has initiated a process by which its software products are carefully licensed and monitored. Most Ixia software applications are controlled by a new License Management process. After a license is purchased, e-mail messages are sent from Ixia to the Lab Manager containing a registration number and password. The contents of that message are then used to obtain a license file for an online Ixia server. The license must be placed either on the workstation or chassis using the licensed software, or on a central license management server.

This document provides information to help determine how licenses are installed and used.

Licensing Installation Overview

The license installation is summarized in the following diagram:

Figure A-1. Licensing Installation Overview

The steps in this process are:

  1. The Purchasing Agent orders licenses for software.
  2. Ixia sends an e-mail to the Lab Manager. This e-mail contains a license Registration number and password.
NOTE. If this email needs to be sent to a list of people, it is advisable that the originator of the order creates a mail list and uses this as the point of contact. This allows Ixia to automatically email a group of people.
  1. The Lab Technician installs the Ixia software.
  2. The Lab Technician installs the license, using the e-mail that was received from Ixia as a key ingredient. This process works best if the Internet is available from the installation computer. If this is not the case, the license can still be installed, but a separate computer attached to the Internet is required. The process is covered in Chapter 2, Registering Licenses.

It is important that the Purchasing Agent provides two pieces of information:

    • The Lab Manager's e-mail address, so that the e-mail from Ixia containing the license registration number reaches the correct person.
    • The name of the workstation or chassis for which the license is destined. This will help the Purchasing Agent and the Lab Technician ensure that the appropriate license is used. This information will appear within the e-mail that Purchasing Agent receives.

Note that each license is dedicated to a particular workstation where the software will run or to the Ixia chassis that holds the load modules that are used by the software. The license file itself knows which type it is and the Lab Tech is asked whether to install on the "workstation" or "chassis", as indicated in the license file. After a license is installed, it may only be used for a particular workstation or chassis. A license file may be moved from one workstation or chassis to another by a process known as de-registration. This is covered in Chapter 3, De-registering Licenses.

License File Placement

Although a particular license is dedicated to a specific workstation or chassis, the license file itself must be placed in one of two places. These choices are shown in the figure below.

Figure A-2. License File Placement

The two choices are:

    • Workstation or Ixia Chassis – the workstation where the software will run or on the Ixia chassis that holds the load modules that are used by the software. The workstation/chassis can be the same machine as the Installation Host.
    • License Server – a central Windows-based server that can be used to hold licenses for multiple software products. One or more License Servers may be used within an organization. A central License Server allows for listing, monitoring, and controlling all of the licenses from a single point. The License Server can be the same machine as the Installation Host.

The preferred location is a central License Server, but the following items must be considered in this choice:

  1. The server must be a reliable, always-up machine. If the server is down, all Ixia software products which use the License Server are unable to run.
  2. The server should be backed up on a regular basis.
  3. Ixia license management software must be installed on the server. All current Ixia chassis have this server installed by default and may be used as a License Server. Older chassis may require server installation. If a single chassis is used, and all Ixia applications use load modules on the chassis, then this is an excellent choice as the central License Server.
  4. The IRU must be installed on the License Server. Installing the IRU automatically installs the server portion of the licensing software.
  5. All users of Ixia license managed software need to be aware of the location of the License Server.

The choice of whether to place licenses on the workstation/chassis or on a central License Server must be communicated to the Lab Tech, for use when installing licenses. This could be done by annotating the e-mail that the Lab Manager forwarded to the Lab Tech prior to license installation.

Checklist

The following is a checklist for the Lab Manager to ensure smooth license installation and usage.

Table A-1. Licensing Checklist
P
Checklist Item

 

When ordering.

 

  • Give the purchasing agent an e-mail address.

 

  • For each license, give the purchasing agent the name of the workstation or chassis that the license is for.

 

If the License Server is other than an Ixia chassis.

 

  • Install Ixia license management software on the server.

 

Prior to Ixia software installation.

 

  • Ensure that an e-mail message from Ixia with the license registration number and password has been received.

 

  • Pass the e-mail message along to the Lab Tech, including where the license will be installed – either on the workstation/chassis or on a License Server.

 

Prior to Ixia software usage, if a License Server is used.

 

  • Inform all Ixia software users that when prompted for a license location, they should respond with the name of the License Server.
