Appendix C: Licensing Diagnostic Utility

This appendix provides basic information on using the Licensing Diagnostic Utility. This utility helps the user to troubleshoot and understand licensing installation and registration problems.

If licensing difficulties occur, the diagnostic should be run both from the user's PC (where his/her application is run from) and from the license server where licenses are held (usually an Ixia chassis).

This appendix has the following sections:

    • Opening the Diagnostic Utility on page C-2
    • Diagnosing License Registration on page C-6
    • Diagnosing License Procurement on page C-7

The diagnostic utility helps the user to troubleshoot and understand licensing installation and registration problems. To open this utility, select Tools > Diagnostic in the menu bar, as shown in Figure C-1 on page C-2:

Mailing Diagnostic Logs

After running a diagnostic to determine a licensing problem, it may be necessary to email the information in the logs for the test to Ixia support, or another person within the user's organization.

To send an email with the current diagnostic information, select the Mail Logs button in the Diagnostic dialog, as described in Opening the Diagnostic Utility on page C-2. The MailLogs dialog displays, as shown in MailLogs Dialog on page C-5. The results of the last diagnostic run are copied to the email.

Diagnosing License Registration

If a user cannot register their license, select Register from the Mode section and press Start, as shown in Figure C-4 on page C-6:

Figure C-4. Running a Registration Diagnostic

Nominal output is shown in Figure C-4 above.

If a user cannot register from a PC, the license server is also a viable location to perform a registration. Make sure to try registration from the license server (chassis). If there is a problem, the diagnostic utility should be run in the same manner from there.

Diagnosing License Procurement

If a license cannot be obtained, the following procedure details how to use the diagnostic utility to find the problem:

  1. Select License from the Mode section.
  2. Select Normal from the Level section.
  3. Select the appropriate scenario from the Server section, one of:
    • Chassis locked, chassis is the license server
    • Chassis locked, separate license server
    • PC locked, no license server
    • PC locked, separate license server
  4. Depending on the server scenario selected, enter the Chassis and License Server name or IP address in the fields provided. These fields are activated only if necessary.
  5. Select Start. Figure shows an example set-up:

    6. Figure C-5.

For full diagnostics, the utility should be run from the license server as well.
